Учебный проект на Node.js: система управления заказами и товарами с реальной бизнес-логикой и тестами (Jest).

Товары (ProductService): добавление, удаление, получение по id, обновление остатков, проверка достаточности остатка.
Заказы (OrderService): создание заказа, добавление/удаление позиций, расчёт итоговой суммы, валидация заказа.

Валидация: непустое имя, положительная цена, неотрицательный остаток, проверка существования товара/заказа, проверка остатков при добавлении в заказ.

Структура

/src
  productService.js   — CRUD и валидация товаров
  orderService.js     — заказы, позиции, расчёт суммы, валидация
  index.js            — экспорт сервисов
/tests
  /unit               — юнит-тесты
  /functional         — функциональные тесты 


Запуск тестов

npm install
npm test

Только юнит-тесты:

npm run test:unit

Только функциональные:

npm run test:functional


Реализованные тесты

Юнит-тесты (15)

Файл и что проверяется

| `tests/unit/productService.test.js` | addProduct (null/undefined, пустое имя, цена 0/отрицательная, валидные данные), getProduct (null/undefined id), removeProduct (ошибка, если товар не найден), updateProductStock (ошибка при отрицательном количестве), hasEnoughStock (превышение запрошенного количества над остатком) |
| `tests/unit/orderService.test.js` | addItemToOrder (недостаток остатка, несуществующий заказ, невалидное количество), calculateOrderTotal (расчёт по нескольким позициям), removeItemFromOrder (товар не в заказе), validateOrder (недостаток остатка после изменения stock), getOrder (null/undefined id) |

Функциональные тесты (2)

1. createProducts_createOrder_addItems_calculateTotal_validate_removeItem_shouldReflectCorrectState  
   Полный сценарий: создание товаров → создание заказа → добавление позиций → расчёт суммы → валидация (успех) → удаление позиций → пересчёт суммы.

2. createProduct_addToOrder_reduceStock_validateOrder_shouldReportInsufficientStock  
   Сценарий: товар → добавление в заказ → уменьшение остатка товара → валидация заказа возвращает `valid: false` и сообщения об ошибках.

Используемые техники тестирования

- Граничные значения: цена 0 и отрицательная, количество в заказе 0 и отрицательное, запрос количества больше остатка.
- Эквивалентное разбиение: валидные данные (успешное добавление товара, расчёт суммы), невалидный класс (пустое имя, несуществующий заказ).
- Невалидные данные: пустая строка имени, строка из пробелов, некорректное количество.
- Проверка выброса ошибок: `toThrow()` для несуществующего товара/заказа, недостатка остатка, невалидных аргументов.
- null / undefined: getProduct(id), getOrder(orderId) при `null`/`undefined` возвращают `null`; addProduct(null/undefined) выбрасывает ошибку.

Имена тестов оформлены в формате: functionName_shouldExpectedResult_whenCondition.
